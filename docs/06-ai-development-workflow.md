# 06 - Flujo de desarrollo con IA

## Antes de pedir código
1. Definir el módulo.
2. Definir reglas de negocio aplicables.
3. Definir tablas usadas.
4. Definir permisos.
5. Definir UI esperada.
6. Definir criterios de aceptación.

## Prompt recomendado para nuevas tareas
```text
Lee AGENTS.md y los documentos en /docs relacionados con este módulo.
Implementa [modulo/feature].
Antes de codificar, resume:
- objetivo
- archivos a modificar
- reglas de negocio aplicables
- riesgos
Luego implementa en pasos pequeños y ejecuta validaciones.
```

## Revisión después de cada tarea
La IA debe entregar:
- Archivos modificados.
- Qué se implementó.
- Qué quedó pendiente.
- Cómo probarlo.
- Riesgos o decisiones tomadas.

## Regla de oro
No dejar que la IA avance demasiado sin validación. Trabajar por fases pequeñas.
